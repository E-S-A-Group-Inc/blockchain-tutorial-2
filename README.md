<p>space-vim is a vim distribution for vim plugins and resources, compatible with Vim and NeoVim.</p>
<p>It is inspired by <a href="https://github.com/syl20bnr/spacemacs">spacemacs</a> and mimics spacemacs in a high level, especially in the whole architecture, key bindings and GUI. if have ever tried spacemacs, you will find space-vim is very similar to it in user experience.</p>
<p>Elegance here means pleasing, graceful as well as simple. If you are unfamiliar with spacemacs, you can visit <a href="http://spacemacs.org/doc/DOCUMENTATION.html" rel="nofollow">spacemacs.org</a> for more about the principle behind that, which is also what space-vim seeks.</p>
<p>The distribution is completely customizable using <code>.spacevim</code>, which is equivalent to <code>.spacemacs</code> in spacemacs.</p>

<p>You can use <code>.spacevim</code> in your home directory to customize space-vim, where you can enable the existing layers, add your extra plugins and private configurations.</p>
<p>If <code>.spacevim</code> does not exist, vanilla vim will be loaded! Refer to <a href="/liuchengxu/space-vim/blob/master/init.spacevim">init.spacevim</a> as an example.</p>
<h3><a id="user-content-presetting" class="anchor" aria-hidden="true" href="#presetting"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Presetting</h3>
<div class="highlight highlight-source-viml"><pre><span class="pl-c"><span class="pl-c">"</span> Comment the following line if you don't want Vim and NeoVim to share the</span>
<span class="pl-c"><span class="pl-c">"</span> same plugin download directory.</span>
<span class="pl-k">let</span> <span class="pl-smi"><span class="pl-k">g:</span>spacevim_plug_home</span> <span class="pl-k">=</span> <span class="pl-s"><span class="pl-pds">'</span>~/.vim/plugged<span class="pl-pds">'</span></span>

<span class="pl-c"><span class="pl-c">"</span> Uncomment the following if you have some nerd font installed.</span>

<p>Basically, <code>g:spacevim_layers</code> almost takes the place of <code>Layer</code> command. As far as I known, most people never use the option of <code>Layer</code> command, e.g., <code>exclude</code>, so <code>g:spacevim_layers</code> could save a lengthy <code>Layer</code> list, requiring less ceremony. Nevertheless, <code>Layer</code> command is still avaiable for some advanced uses.</p>

<p>If have a heavy customized configuration, you can organize them in <em>private</em> directory with <code>packages.vim</code> and <code>config.vim</code> too, which will be loaded on startup. The <em>private</em> directory can be considered as either a single layer, i.e., in which you can put packages.vim and config.vim, or a set of multiple layers.</p>
<h2><a id="user-content-how-to-use" class="anchor" aria-hidden="true" href="#how-to-use"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>How to use</h2>
<p>First of all, I recommend you to look through the existing key bindings via <kbd>SPC ?</kbd>. What's more, you definitely can not miss reading the README of <a href="https://github.com/liuchengxu/space-vim/blob/master/layers/%2Bdistributions/better-defaults/README.md">better-defaults layer</a>, which is of great importance for you to get started quickly.</p>
<p>For detailed instruction, please refer to the README under the certain layer enabled, or you can see config.vim and packages.vim directly.</p>
<p>If the README is not elaborate, sorry for that, space-vim now is in the early stages and a ton of stuff are waiting to be done.</p>
<h3><a id="user-content-bootstrap" class="anchor" aria-hidden="true" href="#bootstrap"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Bootstrap</h3>
<p>The modular design is originally from <a href="https://github.com/syl20bnr/spacemacs">spacemacs</a>. The implementation of logic in space-vim is similar to <a href="https://github.com/junegunn/vim-plug">vim-plug</a>.</p>
<p>If you want to know more about the bootstrap of space-vim, please see <a href="https://github.com/liuchengxu/space-vim/wiki/Bootstrap">bootstrap</a> in wiki.</p>

<h4><a id="user-content-enable-gui-color-in-terminal-vim" class="anchor" aria-hidden="true" href="#enable-gui-color-in-terminal-vim"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Enable GUI color in terminal vim</h4>

<span class="pl-k">let</span> <span class="pl-smi"><span class="pl-k">g:</span>spacevim_enable_true_color</span> <span class="pl-k">=</span> <span class="pl-c1">1</span></pre></div>

<h5><a id="user-content-terminal" class="anchor" aria-hidden="true" href="#terminal"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Terminal</h5>
<p><a target="_blank" rel="noopener noreferrer" href="https://github.com/liuchengxu/space-vim/blob/gh-pages/docs/img/iterm2_powerline_setting.png?raw=true"><img src="https://github.com/liuchengxu/space-vim/raw/gh-pages/docs/img/iterm2_powerline_setting.png?raw=true" align="right" width="550px" alt="iterm2-font-setting" style="max-width:100%;"></a></p>
<p>First, install the <a href="https://github.com/adobe-fonts/source-code-pro">Source Code Pro</a> or <a href="https://github.com/powerline/fonts">Powerline</a> font, which is important for <a href="https://github.com/liuchengxu/space-vim/blob/master/layers/%2Bthemes/airline/README.md">airline</a> layer.</p>

<div class="highlight highlight-source-shell"><pre>$ <span class="pl-c1">cd</span> path/to/space-vim
$ make update</pre></div>

<div class="highlight highlight-source-shell"><pre>$ <span class="pl-c1">cd</span> path/to/space-vim
$ git pull origin master</pre></div>
